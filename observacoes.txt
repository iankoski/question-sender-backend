====================================== ANÁLISE ======================================

Criar as apis com os cadastros de company (empresa), /*customer (cliente),*/ question (pergunta), alternative (alternativa), answer (resposta).

/*Automaticamente um novo usuário criado é um cliente, que pode cadastrar empresas.*/

O sistema deve permitir uma empresa cadastrar perguntas e visualizá-las.

As perguntas devem ser cadastradas com uma data de início e data fim em que elas podem ser respondidas.

O sistema deverá ter uma função para totalizar as respostas de cada pergunta.

Deverá também ter uma página no front end onde o cliente possa visualizar os totais. Esse totalizador também deverá ter um tempo de vida, ou seja, quantos
dias essa visualização de totais ficará disponível para o usuário ver. Pode-se usar o mesmo tempo de vida da pergunta.

O sistema deve permitir a empresa visualizar as respostas de suas perguntas.

O sistema deve permitir que o cliente responda perguntas, acessadas através de leitura de qr code. 

/*A localização do cliente deverá ser verificada sempre que ele fizer login no sistema.

Quando fizer login, ele poderá escolher as empresas cadastradas próximas a ele, e responder as perguntas.*/

Uma pergunta somente pode ser respondida uma vez por cliente. Deverá ser verificado o ID do dispositivo e gravado junto à resposta.
